# **SUBIT‑64 Axis Patterns**

This document catalogs common axis configurations (“patterns”) that appear across the SUBIT‑64 semantic space.  
Patterns are **emergent combinations** of the six binary axes.  
They are not nodes, layers, or transitions — they are **structural motifs** that recur across the system.

This document is **informative**, not normative.

---

# **1. Purpose of Axis Patterns**

Axis patterns help identify:

- semantic families  
- behavioral modes  
- structural tendencies  
- cluster identities  
- trajectory directions  

Patterns are defined purely by **bitwise structure**, not by layer or node semantics.

---

# **2. Pattern Types**

SUBIT‑64 supports five major pattern types:

1. **Binary polarity patterns**  
2. **Two‑axis motifs**  
3. **Three‑axis modes**  
4. **Four‑axis configurations**  
5. **Global patterns (5–6 axes)**  

Each type captures a different level of structural coherence.

---
